# AI-Powered Video Quiz App

An AI-powered quiz application that generates interactive quizzes from video transcripts, allowing users to test their knowledge while having direct access to the source video with timestamp-linked answers.

## Current Status: MVP Complete

The application is fully functional with core features implemented. Users can upload local videos with pre-generated transcripts, customize quiz parameters, take interactive quizzes with video playback, and export results to Anki for spaced repetition learning.

---

## Core Features

### 1. Video & Transcript Input
- **Local video file upload** - Supports MP4, WebM, OGG, MOV, MKV formats
- **Transcript JSON upload** - Accepts Whisper.cpp output format
- **Drag-and-drop interface** - Separate drop zones for video and transcript files
- **File validation** - MIME type and extension checking with user-friendly errors
- **Object URL playback** - Videos play directly from browser memory (no server upload)

### 2. Quiz Customization
- **Question count** - User selects 1-20 questions per quiz
- **Difficulty levels** - Easy, Medium, Hard, or Mixed
- **Custom instructions** - Free-form text to guide AI generation (e.g., "focus on technical concepts", "avoid jargon")

### 3. AI Quiz Generation
- **Claude API integration** - Uses claude-sonnet for question generation
- **Structured output** - JSON format with questions, options, explanations, and timestamps
- **Transcript-grounded** - All questions reference specific parts of the video content
- **Timestamp mapping** - Each correct answer links to the relevant video segment

### 4. Interactive Quiz Interface
- **Flashcard-style display** - One question at a time for focused learning
- **Multiple choice** - 4 options per question (A, B, C, D)
- **Immediate feedback** - Visual indication of correct/incorrect answers
- **Progressive disclosure** - Timestamps shown in explanation section after answering (prevents spoilers)
- **Progress tracking** - Visual progress bar and question counter

### 5. Video Player Integration
- **Split-screen layout** - Video on left, quiz on right
- **Native HTML5 player** - Standard controls with programmatic seeking
- **Timestamp navigation** - Click timestamps to jump to relevant video sections
- **Synchronized playback** - Video seeks automatically when timestamp badges are clicked

### 6. Results & Export
- **Score summary** - Correct/total with percentage
- **Performance messages** - Contextual feedback based on score
- **Review mode** - Navigate back through all questions with answers visible
- **Anki export** - Download quiz as `.txt` file for Anki import
  - Format: `front;back;source`
  - Includes video title and timestamps as source reference

## Technical Architecture

### Tech Stack
| Layer | Technology | Purpose |
|-------|------------|---------|
| Framework | Next.js 16 (App Router) | Server components, API routes, routing |
| Runtime | React 19 | UI components with latest features |
| State | Zustand 5 | Lightweight global state management |
| Styling | Tailwind CSS 4 | Utility-first styling |
| UI Components | Shadcn UI + Radix | Accessible component primitives |
| AI | Anthropic Claude API | Quiz question generation |
| Video | Native HTML5 `<video>` | Simple, dependency-free playback |

### Data Models

```javascript
// Transcript Segment (parsed from Whisper.cpp JSON)
{
  id: string,           // "segment-0", "segment-1", etc.
  text: string,         // Spoken text content
  startTime: number,    // Start time in seconds
  endTime: number       // End time in seconds
}

// Quiz Question (generated by AI)
{
  id: string,           // Unique question identifier
  question: string,     // Question text
  options: [
    {
      id: string,       // "a", "b", "c", "d"
      text: string,     // Answer option text
      isCorrect: boolean,
      timestamp: {
        start: number,  // Seconds
        end: number,    // Seconds
        label: string   // Formatted "2:34"
      }
    }
  ],
  explanation: string   // Why the correct answer is correct
}

// Video Store State
{
  videoFile: File | null,
  videoUrl: string | null,      // Object URL for playback
  videoTitle: string,
  transcript: TranscriptSegment[],
  transcriptFile: File | null,
  isPlaying: boolean,
  currentTime: number,
  duration: number,
  isReady: boolean,
  playerRef: { seekTo: Function }
}

// Quiz Store State
{
  sessionId: string,
  transcript: TranscriptSegment[],
  questions: QuizQuestion[],
  currentQuestionIndex: number,
  answers: { [questionId]: selectedOptionId },
  isComplete: boolean,
  isGeneratingQuiz: boolean,
  error: string | null
}
```

## API Endpoints

### POST /api/generate-quiz

Generates quiz questions from a transcript using Claude AI.

**Request:**
```json
{
  "transcript": [
    { "id": "segment-0", "text": "...", "startTime": 0, "endTime": 8 }
  ],
  "numQuestions": 5,
  "difficulty": "mixed",
  "customInstructions": "Focus on practical examples"
}
```

**Response:**
```json
{
  "questions": [
    {
      "id": "q-abc123",
      "question": "What is the main benefit of...?",
      "options": [...],
      "explanation": "..."
    }
  ]
}
```

### POST /api/transcribe (Legacy)

Original Whisper API transcription endpoint. Deprecated in favor of client-side transcript upload.

## Whisper.cpp Transcript Format

The app expects transcripts in Whisper.cpp JSON format:

```json
{
  "transcription": [
    {
      "timestamps": {
        "from": "00:00:00,000",
        "to": "00:00:08,320"
      },
      "offsets": {
        "from": 0,
        "to": 8320
      },
      "text": " Welcome to today's tutorial..."
    }
  ]
}
```

Generate transcripts locally using:
```bash
whisper.cpp/main -m models/ggml-base.bin -f video.wav -oj
```

Or use the `transcribe` bash alias if configured.

## Known Limitations

1. **No YouTube support** - Must upload local video files
2. **No server-side storage** - All data lives in browser memory
3. **Single session** - Refreshing the page loses quiz progress
4. **English transcripts** - AI prompt optimized for English content
5. **Video file size** - Limited by browser memory (practical limit ~1-2GB)
